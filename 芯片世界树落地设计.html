芯片世界树 · 落地设计方案

核心装置 · 1F具身智能区正中央 · 空间智能的可视化中枢

1. 定位：不是"摘苹果的装置"，是"空间神经系统的可视化"

世界树的核心价值不是采摘互动本身，而是让访客直觉感受到空间在"思考"。

旧叙事 新叙事
机器人从树上摘苹果给你 整个展厅通过树感知你的到来，全场响应
展示"机械臂很准" 展示"SesameX芯片是这个空间的神经系统"
采摘是主角 采摘是末端动作，树本身是主角

APEC级别的wow点：领导团进门，世界树从根部爆发式点亮，光流加速涌向树冠，全场7块LED屏同步切换主题——空间在0.1秒内感受到了他们的到来。

2. 装置结构（5层）

层级 名称 内容 技术
Layer 0 种子 华山A2000芯片实物，心跳脉冲光 LED点光源，常态0.8Hz心跳
Layer 1 树根 发光地台，电路纹理+芯片铭牌 地面LED透光地台，电路蚀刻
Layer 2 树干 世界树光柱，芯片粒子不断涌现 LED柱体+内藏供果机构
Layer 3 枝叶 机器人形态全息粒子 透明屏/投影，随状态切换
Layer 4 树冠 天花板光纤星空 光纤阵列，随触发事件脉冲

装置总高：3.0m（含顶部光球） 占地：直径约1.5m圆形基台 位置：1F具身智能区正中央

3. 光效状态机（核心落地要求）

关键约束：世界树光效必须由软件实时驱动，不能是死循环动画。 展装公司交付时需提供外控接口（DMX512 或 HTTP API），否则空间智能效果无法实现。

3.1 四种光效状态

状态 触发条件 光效表现 持续时长
待机 无访客互动 低频蓝紫色心跳脉冲，0.8Hz，从树根向上缓慢涌动 循环
感知 有访客进入展厅（红外触发） 脉冲加快至2Hz，亮度+30%，颜色偏青白 持续有人期间
VIP响应 VIP签到（RFID触发） 从根部爆发式白光→紫色光流快速涌向树冠，0.5秒完成，然后稳定在高亮紫色 导览全程
采摘激活 访客触发采摘互动 切换D435i视觉视角（树干LED显示识别框画面），绿色脉冲追踪苹果槽位 采摘全程约10s

3.2 与全场联动时序

VIP进门 T+0s
  → 红外感知（本地，<50ms）→ 世界树：爆发式点亮（DMX512本地，<100ms）
  → LED全场主题切换（WebSocket本地，<200ms）
  ↓
T+2s
  → RFID签到确认
  → HTTP→优必选云→Walker S3开始移动（2~5秒延迟可接受，光效已先给wow）
  ↓
T+10s
  → Walker到达，开口致辞
  → 其他机器人各自继续工作（M20巡检、遨博i5待机、咖啡机运转）
  → 体现：这是一个系统在运转，不是表演

关键设计原则：光和屏在第0.1秒给震撼，机器人走过来是叙事推进，2~5秒延迟不是问题。

4. 供果与采摘机构（工程细节）

4.1 供果结构

项目 方案
槽位数量 树干内6个V型出口槽位（A~F），苹果随机落入其中一个
随机机制 供果管顶部物理偏转片，纯机械随机，不依赖软件
槽位高度 距地1.0~1.2m（遨博i5臂展886mm，基座高0.4m，可达范围内）
固定方式 真苹果置于V型槽，重力固定，无需磁铁
补给机构 树干内链条提升机，底部0.5m处加料口，每日操作约2分钟
容量 底部储果仓6~12个，配合每日3次补给（10:00/14:00/17:00）
水果规格 真苹果，直径70~80mm，重量150~180g

4.2 遨博i5协作臂

项目 规格
型号 遨博i5，5轴协作臂，负载5kg，臂展886mm
安装位置 世界树装置右侧0.3m，基座高0.4m
末端执行器 真空吸盘（3~4硅胶吸盘+12V微型真空泵），不损伤果肉
视觉 末端安装RealSense D435i深度相机
识别算法 SesameX Aura端侧，HSV颜色分割→6DoF位姿估计，<200ms

4.3 抓取全流程（总约10秒）

① 供果（自动）    链条提升→偏转片随机落入某槽位
② 视觉扫描 0.5s  D435i扫描6槽，Aura HSV识别→6DoF位姿
③ 运动规划 0.3s  OMPL/RRT路径规划，避开槽位边缘
④ 接近吸附 2.0s  速度<0.1m/s，真空泵启动，吸盘接触苹果
⑤ 提起     1.0s  向上提起，重力脱离V型槽，负压确认
⑥ 递送     3.0s  移动到取物台上方，释放真空，15°斜面滑落
⑦ 复位     2.0s  机械臂回扫描待机位，管道自动补位下一苹果

4.4 视觉环境控制

问题 方案
展厅LED杂光干扰 采摘区上方安装黑色亚克力遮光罩（半圆，隔离杂光）
补光 遮光罩内5500K恒定LED条灯，色温稳定
识别成功率目标 >90%（随机位置+恒定光照+颜色分割）

7. 展装公司接口需求（必须在合同中确认）

这是整个"空间智能"能否落地的关键，在方案定稿前必须谈定。

7.1 必须提供的接口

接口 协议 用途 备注
灯光外控接口 DMX512 或 HTTP API 软件触发光效状态切换 必须支持至少4种预设场景切换
供果机构控制接口 RS232 / 继电器 / HTTP 触发下一个苹果供给 采摘完成后软件自动触发补果
装置状态反馈 MQTT 或 HTTP 上报槽位是否有苹果、机构是否异常 M20巡检时需读取

7.2 确认问题清单（发给展装公司）

1. 世界树灯光是否支持外部控制？用什么协议（DMX512/HTTP/串口）？
2. 是否可以设置至少4种灯效预设，由软件切换？
3. 供果机构是否有电气控制接口，还是纯手动？
4. 树干LED柱体是否可以接入自定义视频内容（如D435i实时画面）？
5. 装置完工后，是否提供接口文档和调试支持？

7.3 如果展装公司不提供接口

降级方案：世界树光效改为定时触发（开馆时固定时间点亮），放弃实时感知联动。

这种情况下"空间智能"仍可通过LED屏内容实时切换来体现，只是少了世界树这个最强的视觉震撼点。

15. 与空间智能OTA的契合度分析与进化路径

15.1 当前契合度评估

世界树的叙事是 L4 级的，但落地技术是 L2 级的。要真正契合 OTA 叙事，需在硬件选型阶段就把"可进化"考虑进去。

OTA层级 世界树应呈现 当前落地 契合度
L2 · 规则联动 预设规则触发（DMX512 4种灯效+定时切换） ✅ 4种光效状态机+DMX512外控 9/10
L3 · 主动感知 VLA感知→空间主动响应（情绪/行为分析驱动光效） ⚠️ 有红外"感知"状态，但只是"人来变亮"，缺行为理解 4/10
L4 · 空间智能 全场传感器融合→自主决策→OTA持续学习→光效"进化" ❌ 固定4种模式循环，无扩展路径 2/10

15.2 三个核心缺口

缺口1：没有"进化"的视觉证据

OTA核心承诺是"展厅会越来越好"。但当前设计L2上线时光效有4种，半年后还是这4种，访客看不到"进化"。

缺口2：缺少"数据→光效"闭环

当前是单向触发（传感器→灯光响应），OTA叙事需要闭环（访客数据积累→云端分析→发现新pattern→OTA下发新光效→世界树"学会了"新的表达方式）。

缺口3：缺少"L4空间智能"的物理载体

L4承诺"空间本身就是机器人"。但世界树目前只是被动响应的灯光装置，没有体现空间对人群密度的理解、对时间的记忆、对事件的记忆。

15.3 六项进化改进

# 改进 具体做法 对应OTA 优先级
1 光效版本号 地台LED显示 v2.1.0，每次OTA更新递增，访客看到"这个空间在升级" L2→L3→L4 P0
2 OTA光效包接口 灯光控制器支持 HTTP API 动态加载新场景JSON（不只DMX512预设），为持续进化留通道 L3 P0
3 访客数据→光效个性化 接入人脸年龄/性别/情绪→生成"专属光效"（年轻人→炫酷青色；老人→温暖橙色） L3 P1
4 空间记忆可视化 地台投影"今日N位访客的光点足迹"，每天结束时世界树"回顾"当天 L4 P1
5 季节性进化 每季度OTA一次"季节光效包"（春绿/夏蓝/秋金/冬紫），让访客每次来都不一样 L4 P2
6 灯光AI模型 L4阶段在边缘服务器训练"光效生成模型"，根据实时传感器数据自动生成新光效 L4 P2

15.4 展装公司必须增加的接口（进化保障）

以下接口必须在合同中确认，否则世界树无法实现 OTA 进化。
