九、机器人状态机与唤醒协议

9.1 统一状态机模型

每个机器人遵循统一的状态流转：

SLEEP（闭馆断电）
  ↓ 开馆上电（08:30）
STANDBY（待机展示态）
  ↓ 唤醒触发
IDLE（就绪态·自检完成·等待指令）
  ↓ DAG分配任务
ACTIVE（执行态·正在执行DAG序列）
  ↓ 任务完成 / 超时 / 故障
RETURN（返回态·回充电桩/待机位）
  ↓ 到达待机位
STANDBY（待机展示态）
  ↓ 闭馆（18:00）
SLEEP

UP（清洁舱） 停在充电桩上（配送舱已换为清洁舱） 充电桩指示灯绿色常亮 DAG调度（M20 UDP检测洒漏→DAG→REST move-task）
i5 臂收拢到待机位（世界树右侧），吸盘松开 世界树LED显示绿色扫描脉冲 触摸按钮（世界树旁触摸屏→DAG→SDK执行序列）
咖啡机 屏保播放"SesameX咖啡待机"动画 取餐屏显示欢迎语 DAG触发（iPad拍照→StyleGAN→4.12上传→4.5下单）
海纳 臂收拢+底盘锁定在展位 展位灯效呼吸 Phase 2（8月后底盘上云→MCP→DAG）

9.3 唤醒协议详解

C1唤醒链路

访客NFC签到台刷RFID手环
  → NFC读卡器读取VIP ID（<0.5秒）
  → 本地服务器SDK接收（WiFi TCP/IP）
  → DAG引擎收到"NFC_SIGN_IN"事件
  → DAG查询VIP信息（姓名/职务）
  → DAG下发任务到C1云平台
  → C1从STANDBY→IDLE（自检关节+启动语音模块）
  → C1从IDLE→ACTIVE（走向入口，开始边走边讲）
  → 品牌墙LED同步切换"欢迎[姓名/职务]"

M20唤醒链路

方式A（定时巡逻）：
  DAG定时器每30分钟触发
  → UDP指令下发"M20_PATROL_START"
  → M20从STANDBY→IDLE（关节解锁+LiDAR启动）
  → M20从IDLE→ACTIVE（自主建图+按预设点位巡逻）
  → 巡逻完成后→RETURN（回跑道起点趴伏）→STANDBY

方式B（越障表演）：
  DAG在VIP场景T+90s触发
  → UDP指令"M20_PERFORMANCE_START"
  → M20执行越障序列（22cm台阶/80cm高台）
  → 完成后→RETURN→STANDBY

方式C（洒漏检测）：
  M20在ACTIVE巡逻态时热成像检测到温差>2°C
  → UDP上报"SPILL_DETECTED + 坐标"
  → DAG收到→触发洒漏自愈DAG序列

UP唤醒链路（送物）

访客在入口包架放置物品
  → RFID传感器检测物品放置
  → DAG收到"ITEM_PLACED"事件
  → DAG调用云迹REST API: transport
    {robotId, cabinId, via:{point:"包架"}, target:{point:"会议室"}, autoBack:true}
  → UP从STANDBY（充电桩）→IDLE→ACTIVE
  → UP到包架取物→走廊导航→到达会议室
  → notification通知（短信/VMS）
  → 访客二维码取物→UP→RETURN（autoBack回充）→STANDBY

i5唤醒链路（采摘）

访客在世界树旁触摸屏点击"开始采摘"（或语音"我要摘苹果"）
  → 触摸按钮信号→DAG
  → DAG下发执行序列到遨博云平台
  → 云平台→i5 SDK:
    Step1: 链条提升机供果（物理触发，2秒）
    Step2: D435i扫描6槽HSV识别（SDK获取臂数据，0.5秒）
    Step3: OMPL路径规划→臂移动到苹果位姿（SDK执行序列，2秒）
    Step4: 真空泵启动→吸盘吸附（SDK控制IO，2秒）
    Step5: 臂提起→移动到递送位→释放真空（SDK执行序列，3秒）
    Step6: 臂回到待机位（SDK执行序列，2秒）
  → 全流程约10秒→i5回到STANDBY
  → 世界树LED从"采摘中"红色→恢复绿色扫描脉冲
